# 数据来源边界说明

## 目的

明确当前项目中哪些分析来自真实工时数据，哪些内容仍是模拟扩展位，避免把示意图误读为真实业务结论。

## 当前真实来源

- 工时原始数据
- 员工日聚合结果
- 任务明细
- 核验状态
- 基于真实工时的规则推导指标

以上内容可用于：

- 观察工时分布和趋势
- 识别异常员工日
- 分析项目投入结构
- 生成基于规则的风险提示

以上内容不应直接用于：

- 员工绩效排名
- 跨团队强比较
- 因果判断

## 当前模拟来源

- Git
- AI 使用
- 用户反馈

这些数据目前仅用于：

- 验证页面结构与图面布局
- 预留后续多源协同分析入口
- 提前演练指标口径和交互方式

这些数据目前不应用于：

- 输出真实研发效能结论
- 做项目优先级决策
- 对员工或项目做正式评价

## 页面展示规则

- 页面顶部应明确展示真实来源和模拟来源边界
- 依赖模拟数据的图表必须带统一标识
- 报告页和导出内容必须保留边界说明

## 后续接入原则

- 任一模拟来源接入真实接口后，需要同步更新页面说明
- 指标口径文档和来源边界文档要一起更新
- 若真实接入范围不完整，应继续标注“部分真实、部分模拟”
