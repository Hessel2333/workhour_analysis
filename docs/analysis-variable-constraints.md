# 分析变量关系约束规范

## 目的

这份规范用于约束本项目中的相关性分析、回归分析、聚类分析与解释性图表，避免把构成项重新拿去解释其自身派生指标，导致图表在形式上成立、但在方法论上失真。

适用范围：

- [相关性页](/Users/tian/Codes/workhour_analysis/src/pages/CorrelationPage.tsx)
- 报告页中的统计解释图
- 后续新增的 Python / 本地规则分析图

## 变量分类

### 1. 直接观测变量

定义：直接来自原始工时明细或员工日聚合，不依赖规则标签或复合评分。

示例：

- 单日工时 `reportHour`
- 当天任务数 `taskCount`
- 当天项目数 `projectCount`
- 核验工时 `verifyHour`
- 未核验标记 `verifyHour === 0 && reportHour > 0`
- 原始任务文本 `taskName`
- 项目日工时序列

允许用途：

- 相关性
- 分布对比
- 聚类
- 变点检测
- 文本分析

### 2. 派生指标

定义：由直接观测变量计算而来，但本身不是规则告警分，也不是模型输出。

示例：

- 日均工时
- 日均任务数
- 日均项目数
- 多项目率
- 集中度
- 核验缺口率
- 返工占比
- 工时波动率

允许用途：

- 可参与相关性、聚类、排序、分组比较
- 但必须明确公式来源

限制：

- 若指标 A 的公式直接包含指标 B，则禁止把 `A ~ B` 包装为“独立统计关系”
- 若两个派生指标共享大量同一底层变量，图表说明中必须提示“口径存在耦合”

### 3. 规则型告警指标

定义：由若干规则条件组合得到，用于筛查、排序、告警。

示例：

- 异常日 `isAnomalous`
- 异常分 `anomalyScore`
- 异常日占比
- 风险分
- 数据健康分中的规则子项

允许用途：

- 预警
- 排序
- 分层筛查
- 规则触发说明

禁止用途：

- 作为被解释变量进入相关性、回归、偏相关、驱动因素分析
- 与其自身构成项做“发现型统计关系”展示

### 4. 模型输出 / 模拟输出

定义：由聚类、预测、评分模型或 mock 数据生成的结果。

示例：

- 聚类标签
- 预测风险
- AI 深度分
- mock 反馈指标

允许用途：

- 辅助探索
- 分类展示
- 模型对比

限制：

- 必须单独标记来源
- 不得与真实直接观测变量混写为“事实结论”

## 硬性规则

### 规则 1

禁止将某指标的构成项作为自变量，再对该指标本身做相关性、回归或偏相关解释。

错误示例：

- `任务数 -> 异常日占比`
- `项目数 -> 异常分`
- `未核验率 -> 异常日`

原因：

这些变量已经直接参与了异常指标的定义，不具备独立解释意义。

### 规则 2

规则型告警指标默认只能用于：

- 触发说明
- 排序
- 筛查
- 看板提醒

不能用于“驱动因素发现型”图表。

### 规则 3

相关性页和其他分析页优先展示两类关系：

- 直接观测变量之间
- 公式无直接包含关系的派生指标之间

### 规则 4

若变量之间存在明显公式耦合，必须至少满足一条：

- 改成构成说明图，而不是解释型统计图
- 在图表说明中明确标注“定义相关，非独立关系”
- 直接移除该图

### 规则 5

聚类分析优先使用：

- 直接观测变量
- 低耦合派生指标

禁止把规则告警结果本身作为核心聚类特征。

## 本项目中的明确约束

### 当前不应作为被解释对象的变量

- `isAnomalous`
- `anomalyScore`
- `异常日占比`

原因：

它们直接由以下规则项构成：

- 单日工时是否过高
- 当天任务数是否过多
- 当天项目数是否过多
- 是否未核验

### 当前更适合继续分析的目标

- 返工占比
- 工时波动率
- 核验缺口率
- 项目变点
- 关键词结构
- 员工直接画像聚类

## 分析图执行规则

分析图中的图表默认遵循以下顺序：

1. 优先直接因素图
2. 其次低耦合派生指标图
3. 规则型指标只做告警说明，不做驱动解释
4. 模型输出必须显式标注

当前已经按本规范调整的图包括：

- 直接因素相关热力图
- 项目工时变点检测
- 员工直接画像聚类图
- 待复核任务关键词频次
- 待复核任务关键词共现网络

## 后续建议

建议后续为每个指标增加 metadata：

- `kind: raw | derived | rule | model`
- `dependsOn: string[]`

在出图前自动做依赖检查，若发现：

- 被解释变量依赖解释变量
- 或两者存在直接公式包含关系

则阻止该图生成，或强制要求标注“定义耦合”。
