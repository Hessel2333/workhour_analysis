# 基于工时分析建议的任务清单

## 目的
- 本文基于当前项目实现情况、根目录研究文档 [deep-research-report_ChatGPT.md](/Users/tian/Codes/workhour_analysis/deep-research-report_ChatGPT.md)、[deep-research-report_Gemini.md](/Users/tian/Codes/workhour_analysis/deep-research-report_Gemini.md)，以及新增这份“工时分析底盘建议”整理。
- 目标不是重复需求文档，而是明确：`当前已覆盖什么`、`还缺什么`、`下一步该做什么`。

## 总体判断
- 当前项目已经具备一套可运行的工时分析底盘，覆盖了基础趋势、项目投入、员工负载、任务主题、核验质量、异常识别、详情下钻等核心能力。
- 但相对建议中的“完整工时分析底盘”，仍有 4 个明显短板：
  1. 基础盘点指标还不够完整，尤其是 `每周 / 每月 / 核验偏差 / 加班天数`。
  2. 工时结构分析已有图，但“任务性质分类体系”还不够贴近软件研发语义。
  3. 效率与负载分析已有雏形，但缺少 `连续高工时 / 返工期 / 项目阶段识别` 这类高价值专题。
  4. 核验质量分析目前更偏“提醒视角”，还没有完整做成老板能直接读懂的核验管理看板。

## 当前已覆盖

### 1. 基础盘点类
- 已有：总工时、活跃员工、项目数、人均工时、总加班、法定假日加班、数据健康分。
- 已有：按 `月份 / 年份` 切换的趋势图，年份模式按月展示，月份模式按天展示。
- 已有：项目总工时、项目参与人数、员工参与项目情况、员工多项目率。
- 已有：核验状态字段解析，未核验提醒、核验覆盖率已进入数据备注和质量提醒。
- 已有：不可能工时值自动识别并默认剔除。

### 2. 工时结构类
- 已有：项目投入结构图、项目占比河流式面积图、工时流向图。
- 已有：员工工作结构图、员工项目参与演进、员工任务类型演进。
- 已有：项目任务类型演进、项目任务主题分布。
- 已有：任务关键词画像、主题分类、主题树图。

### 3. 效率与负载类
- 已有：员工风险分布、员工切换压力、多项目率、集中度、异常员工日。
- 已有：加班变化图、加班重点人图。
- 已有：项目复杂度气泡图、趋势斜率、项目参与面和人均投入分析。
- 已有：员工 / 项目 / 日期 / 任务详情弹窗，下钻能力较完整。

### 4. 核验质量类
- 已有：未核验、缺少明细、不可能工时、待确认任务、未分类任务等质量提醒。
- 已有：数据健康分、质量页、待确认清单、分类复核清单。

### 5. 文本挖掘类
- 已有：规则分类器，已支持 `开发、测试、部署、维护、设计、会议、学习、文档、运营、现场支持、合规管理、待确认、未分类`。
- 已有：关键词画像和人工复核入口。
- 已有：一部分明显可归类任务已补规则，减少了未分类比例。

## 主要缺口

### A. 基础盘点类缺口
- 缺少 `员工月度总工时柱状图` 这一类更直观的老板视图图表。
- 缺少 `员工工时箱线图`，无法快速看团队整体离散程度。
- 缺少 `加班天数 / 连续高工时天数 / 月度加班率`。
- 缺少 `项目人天投入` 这种更管理化的口径。
- 缺少 `核验通过率、未核验工时占比` 的独立可视化表达。

### B. 工时结构类缺口
- 当前任务类型更偏通用分类，缺少更贴近软件研发的二级类：
  - 开发实现
  - 接口对接
  - UI / 页面
  - 测试 / 联调
  - Bug 修复
  - 会议 / 沟通
  - 学习 / 预研
  - 部署 / 发布
  - 文档 / 验收支持
- 缺少 `员工-项目工时热力图`。
- 缺少 `按月份看项目切换` 的专门图表。
- 缺少 `公司整体任务类型画像` 与 `员工任务画像对比`。

### C. 效率与负载类缺口
- 缺少 `员工连续高工时天数`。
- 缺少 `项目冲刺期识别`。
- 缺少 `返工型工时识别`：
  - 修改
  - bug
  - 测试修改
  - 客户反馈调整
- 缺少 `救火型员工识别`：
  - 高切换
  - 高支持类任务
  - 高频跨项目
- 缺少 `项目四象限图：总工时 vs 修改类工时占比`。

### D. 核验质量类缺口
- 缺少 `填报工时 vs 核验工时偏差` 的系统分析。
- 缺少：
  - 员工核验通过率
  - 项目核验通过率
  - 月度未核验工时趋势
  - 高风险项目未核验积压排行
- 目前质量页更偏“问题清单”，缺少面向老板的“核验管理看板”。

### E. 文本挖掘类缺口
- 缺少 `返工特征识别词典`。
- 缺少 `项目阶段识别`：
  - 需求 / 设计阶段
  - 开发阶段
  - 联调阶段
  - 测试阶段
  - 上线阶段
  - 维护阶段
- 缺少 `AI 相关研发` 作为单独主题。
- 缺少 `客户反馈调整` 作为单独返工类标签。
- 缺少文本规则解释页，无法明确说明某条任务为什么被归到某类。
